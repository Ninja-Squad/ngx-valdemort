Features

  • Minimum markup needed
  • Consistent, default error messages
  • Generic or control-specific default error messages by specifying a label
  • Overriding of default messages for custom error messages
  • All error messages have access to the error value (min, max, required length, etc.)
  • I18n possible
  • HTML in messages possible
  • Use of pipes in messages possible
  • Usable in reactive forms, template-driven forms, or standalone controls
  • Usable on controls nested in form groups or form arrays
  • Central configuration service
  • Free, and open-source. The code is available on Github
